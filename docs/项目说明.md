## 第 1 页：项目基本信息

项目名称：FZU Online Note

学号：待填写

运行方式：直接打开仓库根目录下的 `index.html`，或在项目根目录执行 `python -m http.server 4173` 后访问 `http://127.0.0.1:4173/index.html`。

本项目是一个轻量级在线笔记 Web 应用，目标是用 HTML、CSS 和 JavaScript 完成一个可以真实使用的笔记管理页面。应用围绕“记录、查找、整理”三个核心场景设计：用户可以新增课程笔记、编辑已有内容、删除不再需要的记录，并通过搜索、筛选、排序等方式快速定位笔记。

项目采用单页面多模块结构，页面从上到下分为笔记编辑区、笔记库、角色主题图库和数据概览。编辑区用于输入和修改笔记，笔记库用于展示和管理已有笔记，图库用于展示主题角色，数据概览用于辅助观察当前笔记数量、标签数量和今日新增数量。

## 第 2 页：需求对应与技术选型

课程基本要求包括 HTML 语义化结构、CSS 合理布局、JavaScript 表单校验、动态渲染、数据持久化和完整的数据管理能力。本项目选择“在线笔记”作为选题，采用基础版技术路线：HTML + CSS + 原生 JavaScript + localStorage。

HTML 部分使用 `header`、`main`、`section`、`form`、`template` 等语义化标签。页面不是简单堆叠元素，而是按照功能模块划分结构，使编辑区、列表区、图库区和概览区各自职责清晰。

CSS 部分使用 Grid 和 Flex 完成主布局，结合媒体查询适配桌面与移动端。表单输入、按钮悬停、错误提示、置顶状态和卡片边框都提供了可见反馈。整体风格参考甜点咖啡厅与二次元角色图库，颜色上使用粉色、薄荷绿、蜂蜜黄、天空蓝和可可色组合，避免单一色调。

JavaScript 部分负责表单校验、事件监听、数据增删改查、搜索筛选、排序统计和 localStorage 持久化。项目没有引入 Vue、PHP 或数据库，是为了保持运行方式简单，符合课程基础版要求，也方便压缩提交和现场展示。

## 第 3 页：功能设计与交互流程

项目的主要功能围绕笔记 CRUD 展开。新增笔记时，用户需要输入标题和内容，可以选择标签、角色主题、笔记状态，并可勾选置顶。提交表单时，程序会先检查标题和内容是否为空、内容长度是否足够、标签数量是否超过限制，校验通过后才写入数据。

查看笔记时，页面会根据当前数据动态渲染卡片。每张卡片包含角色图、标题、摘要、标签、更新时间和操作按钮。置顶笔记会优先排序，并通过边框颜色和置顶按钮状态体现。

编辑笔记时，点击卡片中的“编辑”按钮，原笔记内容会回填到左侧表单。保存后不会创建新笔记，而是更新原笔记并刷新更新时间。删除笔记时，系统会弹出确认框，避免误删。复制功能用于快速创建相似内容，适合课堂笔记模板复用。

搜索与筛选用于提升使用效率。搜索框可以匹配标题、正文、标签和角色名；筛选按钮支持全部、置顶、今日新增；排序下拉框支持最近更新、创建时间和标题 A-Z。数据概览模块会同步更新总笔记数、置顶数、标签数和今日新增数。

## 第 4 页：数据结构与持久化实现

项目的数据保存在浏览器 localStorage 中，存储键名为 `fzu-online-note:v1`。每条笔记使用一个对象表示，包含 `id`、`title`、`content`、`tags`、`theme`、`mood`、`pinned`、`createdAt` 和 `updatedAt` 等字段。

`id` 用于唯一定位笔记，优先使用 `crypto.randomUUID()` 生成；考虑到部分运行环境可能不支持该 API，代码提供了基于时间戳和随机数的兜底方案。`title` 和 `content` 保存核心文本，`tags` 保存标签数组，`theme` 对应角色主题，`mood` 表示学习、灵感、待办或复盘状态，`pinned` 表示是否置顶。

新增、编辑、删除、复制和置顶操作都会改变内存中的 `notes` 数组，然后调用 `persist()` 写回 localStorage。页面初始化时，`loadNotes()` 会尝试读取本地数据；如果没有数据或解析失败，则使用内置示例笔记作为初始内容。这样第一次打开页面也能看到完整界面，方便展示功能。

localStorage 的优点是实现简单、无需服务器、刷新页面后数据仍然存在。限制是数据只保存在当前浏览器中，清除浏览器数据或更换浏览器后无法同步。对于课程基础版项目，这种方式足够体现数据持久化和前端数据管理能力。

## 第 5 页：源码实现说明

项目源码主要由三个文件构成：`index.html`、`assets/css/styles.css` 和 `assets/js/app.js`。

`index.html` 负责页面结构。表单区域包含标题、内容、标签、主题、状态和置顶开关。列表区域使用 `<template>` 定义笔记卡片模板，JavaScript 渲染时会克隆模板并填充数据。这样的写法减少了在 JS 中拼接大量 HTML 字符串的风险，也让结构更容易维护。

`styles.css` 负责视觉和响应式布局。主工作区使用两列 Grid，左侧为编辑面板，右侧为笔记库；在较窄屏幕下自动切换为单列布局。卡片、按钮、表单和统计模块都设置了固定的间距和边框半径，避免内容变化时出现明显布局跳动。

`app.js` 是核心逻辑文件。`themes` 对象维护角色主题、图片直链、本地回退图、角色信息和强调色；`handleSubmit()` 处理新增与更新；`validateForm()` 处理校验；`renderNotes()` 和 `createNoteCard()` 负责动态渲染；`getVisibleNotes()` 统一处理搜索、筛选与排序；`persist()` 和 `loadNotes()` 负责本地存储。

源码实现中比较重要的一点是状态集中管理。笔记数据始终以 `notes` 数组作为当前状态，所有界面变化都从数据重新渲染出来，避免不同 DOM 节点之间出现状态不一致。

## 第 6 页：遇到的问题与解决方案

第一个问题是文档中的标题层级和项目要求需要转为 Markdown。原始需求文档是 `.docx`，且内部标题样式并不完整。处理时先读取段落内容，再根据编号和项目符号整理成 Markdown 标题、列表和链接，最终保存到 `docs/需求文档.md`。

第二个问题是图片素材的处理。最初考虑把本地 SVG 作为主题图，但后续调整为使用萌娘百科中的《巧克甜恋》角色信息和图片直链。为了避免网络不稳定影响答辩展示，项目保留了 `assets/local-fallback/` 本地原图缓存作为失败回退；同时该目录写入 `.gitignore`，避免把缓存图片提交进仓库。

第三个问题是表单编辑与新增容易混在一起。解决方式是设置隐藏字段 `note-id`：当字段为空时表示新增，当字段有值时表示更新已有笔记。点击“新建”会清空隐藏字段和表单，避免继续覆盖原笔记。

第四个问题是列表排序和筛选逻辑容易分散。项目将搜索、筛选、排序统一放在 `getVisibleNotes()` 中处理：先过滤，再排序，最后渲染。这让交互逻辑更集中，后续增加筛选条件也更方便。

## 第 7 页：项目结构、素材说明与总结

项目结构如下：

```text
FZU-Online-Note/
├── index.html
├── README.md
├── assets/
│   ├── css/
│   │   └── styles.css
│   ├── js/

│   └── local-fallback/  # 本地图片缓存，已写入 .gitignore
└── docs/
    ├── 需求文档.md
    └── 项目说明.md
```

角色主题参考萌娘百科《巧克甜恋》相关页面，页面包含雪村千绘莉、天宫美久栗、御园莓华、舞羽娜娜、百濑辉夜、百濑美月和小花。角色图片默认使用萌娘百科图片直链，本地缓存仅作为答辩时的回退资源，不进入 Git 仓库。

本项目完成了在线笔记的基础功能，实现了表单校验、动态渲染、增删改查、搜索筛选、排序统计和 localStorage 持久化。整体运行方式简单，适合课程作业提交和现场演示。

后续可以继续扩展 Markdown 预览、笔记导入导出、标签管理、云端同步和用户登录。如果进一步使用 Vue 或后端数据库，也可以在当前数据结构基础上继续演进。
