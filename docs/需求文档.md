# Web程序设计实践课程作业

## 1. 课程项目要求

完成一个轻量级 Web 应用程序，综合运用 HTML/CSS/JavaScript 前端核心技术，可选结合 PHP 后端开发或 Vue 前端框架，实现数据管理与交互逻辑，检验对 Web 开发全流程的掌握。

### 1.1 选题方向与要求

从以下场景中选择（或自主设计轻量化应用）：

- 在线备忘录
- 在线博客
- 在线笔记
- 在线记账本等

基本要求（所有选题必做）：

- HTML：
  - 语义化标签使用（header/section/form）
  - 页面结构合理性（至少 2 个页面，或单页面多模块）
- CSS：合理布局，视觉反馈、色彩协调等
- JavaScript：表单校验、动态渲染、数据持久化
- 基本数据管理：增删改查功能完整性、存储实现

### 1.2 项目难度可选

在基础要求之上，同学们可根据自身的情况，自由叠加前端框架（Vue）或后端开发（PHP）的进阶功能。

- 基础版：使用 HTML+CSS 完成界面设计，用原生 JavaScript 实现基础交互（如增删改查），通过 localStorage 完成数据本地存储和管理；
- 可选，前端进阶版（VUE）：用 Vue.js 替代原生 JS 的 DOM 操作（如通过 .vue 组件拆分界面：利用 Vue 的响应式数据绑定简化交互逻辑（如输入框内容自动同步到数据层）
- 可选，后端进阶版（PHP，也可以用别的）：可额外加入 PHP 服务端开发，通过 MySQL 数据库实现数据持久化，提升应用的实用性（如用户登录等）

### 1.3 项目考核（个人项目不分组）

- 将整个项目文件夹（包含所有代码、资源和项目说明文档）压缩成一个 .zip 文件。
- 压缩包命名规范：学号_姓名_项目名称.zip（例如：2023001_张三_在线备忘录.zip）。
- 2026.6.25 前邮件发送至 jiechen202@fzu.edu.cn，邮件标题为《Web_学号_姓名_项目名称》

#### 1.3.1 项目展示

实践课程以项目考核为最终成绩，暂定最后一节实践课当天，同学们进行项目展示与答辩（每个人 5min）

#### 1.3.2 项目代码

- 源代码：所有 HTML, CSS, JavaScript (.js), PHP (.php), Vue (.vue) 等源文件。
- 资源文件：项目中使用的图片、字体、图标等。
- 数据库文件（若使用后端数据库）：数据库导出文件（如 .sql 脚本），包含表结构和必要的初始数据。
- 在 READEME.md 中说明如何导入和配置数据库等。

#### 1.3.3 项目说明文档

除项目工程文件和现场演示外，每位同学需在项目最终提交时，一并提交一份《项目说明文档》（电子版，例如 .docx 或 .pdf 格式）。该文档旨在阐述项目设计思路、评估项目完成度和确认个人贡献。

文档要求：内容清晰、条理分明、语言简洁。适当使用截图辅助说明（如关键代码、数据库设计、复杂界面等），8 页以内，不要卷字数，避免过多无关内容。

文档应至少包含以下内容：

- 项目基本信息：
  - 项目名称、选题方向。作者姓名、学号。
  - 项目访问/运行方式说明（如：GitHub Pages 链接、本地运行步骤、后端部署说明等）。
- 设计与实现概述：
  - 项目简介：简述项目的主要功能和目标。
  - 技术选型：清晰列出所使用的主要技术（如 HTML、CSS、JavaScript、Vue、PHP、MySQL/localStorage 等），并简要说明选择关键技术（尤其是进阶技术）的理由。
  - 核心功能实现：扼要描述主要功能（特别是增删改查、表单校验、动态渲染、数据持久化等）的实现思路和关键逻辑。
  - 遇到的挑战与解决：简述在开发过程中遇到的 1-2 个主要技术难题以及如何解决的。
- 项目结构：
  - 简要说明项目主要的文件夹和文件构成及其用途。
- 学习总结与自我评估：
  - 概述通过此项目的主要学习收获。
  - 对项目完成情况的自我评估，以及（可选）未来可改进或扩展的方向。
  - 原创性声明与引用（可选，但推荐）：简短声明项目为个人独立完成。如使用了重要的第三方库、框架或参考了关键代码片段，请注明来源。
